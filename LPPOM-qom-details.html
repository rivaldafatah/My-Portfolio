QA Testing For LPPOM Digitalization (QOM Module / Apps)

Overview: The Halal Certificate Digital System is a web-based platform developed to manage the process of issuing halal certificates for registered companies and their food products. The system supports the complete certification workflow, starting from product registration, laboratory testing verification, halal audit evaluation, and final certificate issuance once all requirements are fulfilled. Through this platform, authorized users can manage company data, product information, and certification documentation in a structured digital process. The system helps streamline the halal certification workflow, ensuring transparency and proper documentation throughout the evaluation and approval stages. In this project, testing activities included both manual testing and automation testing using Katalon Studio to validate system functionality, workflow processes, and overall system stability.

Testing Types

  • Functional Testing To ensure the software performs the functions it was designed for from the end user's perspective.
  • Regression Testing Ensure code changes do not break existing features.
  • Smoke Testing Quick testing to ensure crucial functions (build) are stable.
  • Integration Testing a software testing method in which various units or modules of an application are combined and tested collectively to ensure that interactions, data flow, and communication between the modules are working correctly.

Result

Automation testing using Katalon Studio was implemented to improve testing efficiency and validate critical system functionalities consistently. As a result, the system achieved improved reliability, smoother workflow execution, and better readiness for supporting the digital halal certification process.
