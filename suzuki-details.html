QA Testing For Traceability Monitoring System

Overview: The Traceability Monitoring System is a system used to check and track the condition of vehicle engines during the production process, from the initial stage to the final stage. This system ensures that each tested engine goes through a series of automated inspection processes to determine whether it meets the established quality standards. The results of the testing process will produce a conclusion in the form of an OK (fit for use) or NG (not fit for use) status. This system is integrated with various technologies, including IoT devices for data collection directly from the engine, a mobile application for scanning and data input in the field, and a web monitoring dashboard that allows operational and management teams to monitor test results, engine status, and traceability history in real-time.

Result

Through comprehensive testing of the Traceability Monitoring System, a total of 533 test cases were created and executed across multiple modules, including IoT data integration, mobile scanning processes, and the web monitoring dashboard. During the testing cycle, 99 bugs were identified and documented, covering functional issues, data synchronization errors, and UI inconsistencies. The testing process helped ensure accurate data flow between IoT devices, the mobile application, and the web dashboard, improving the reliability of the engine inspection workflow. As a result, the system was able to provide more stable monitoring, accurate traceability records, and reliable OK / NG engine validation results for production operations
